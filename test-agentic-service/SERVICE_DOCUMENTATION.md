# 저기어때 - 여행 계획 에이전틱 서비스 상세 문서

## **3. 주요 기능 및 동작 시나리오**

### **3.1 사용자 시나리오**

#### **시나리오 1: 해변 휴양 여행 계획**

**사용자**: 직장인 김미래 (30대, 5일 휴가)

**상황**: 
- 스트레스 해소를 위한 해변 휴양 여행을 원함
- 예산은 200만원 내외
- 서울 출발, 해외 여행지 미정

**서비스 이용 과정**:

1. **입력 단계**
   ```
   👤 사용자 입력:
   - 여행 주제: "해변에서 휴양"
   - 여행 일수: 5일
   - 예산: 2,000,000원
   - 출발 도시: 서울
   - RAG 활성화: ✅
   ```

2. **Agent A: 도시 추천**
   ```
   🌍 도시 추천 에이전트 실행
   
   [웹 검색]
   - 쿼리: "해변에서 휴양 여행 추천 도시 해외"
   - DuckDuckGo에서 실시간 여행 정보 수집
   - 발리, 푸켓, 세부 등 인기 해변 도시 정보 확보
   
   [AI 분석 및 추천]
   - 검색 결과 + LLM 지식 통합
   - 추천 결과:
     1. 발리, 인도네시아 - "완벽한 해변과 힐링 스파가 유명"
     2. 푸켓, 태국 - "저렴한 가격에 아름다운 해변 즐기기"
     3. 세부, 필리핀 - "에메랄드빛 바다와 스노클링 명소"
   
   [자동 선택]
   - 첫 번째 도시 '발리' 자동 선택
   ```

3. **Agent B: 항공권 검색**
   ```
   ✈️ 항공권 검색 에이전트 실행
   
   [웹 검색]
   - 쿼리: "서울 발리 항공권 평균 가격 항공사"
   - 실제 항공권 가격 정보 수집
   - 주요 항공사 및 시즌별 가격 정보 확보
   
   [AI 분석 및 생성]
   - 출발일: 2026-02-26 (오늘 + 30일)
   - 귀국일: 2026-03-03 (출발일 + 5일)
   - 항공권 정보:
     * 항공사: 대한항공
     * 출발 공항: ICN (인천)
     * 도착 공항: DPS (발리)
     * 가격: 850,000원 (왕복)
   ```

4. **Agent C: 일정 + 예산 계획**
   ```
   📅 일정 계획 에이전트 실행
   
   [웹 검색]
   - 쿼리: "발리 해변휴양 여행 일정 추천 명소 맛집"
   - 현지 명소, 맛집, 액티비티 정보 수집
   - 우붓, 스미냑, 울루와투 등 핫플레이스 확보
   
   [AI 분석 및 계획]
   - Day 1: 공항 도착 → 스미냑 해변 산책 → 호텔 체크인
   - Day 2: 우붓 원숭이 숲 → 테갈랄랑 라이스테라스 → 전통 발리 마사지
   - Day 3: 스노클링 투어 → 해변 휴식 → 선셋 감상
   - Day 4: 울루와투 사원 → 케착 댄스 관람 → 징글 해변
   - Day 5: 쇼핑 → 공항 출발
   
   - 예산 분배:
     * 항공권: 850,000원
     * 숙소: 600,000원 (1박 120,000원 × 5박)
     * 식비: 350,000원 (1일 70,000원 × 5일)
     * 기타: 200,000원 (투어, 쇼핑, 교통)
     * 총 예산: 2,000,000원
   ```

5. **결과 확인**
   ```
   📋 완성된 여행 계획
   
   [탭 1: 추천 도시]
   - 발리, 인도네시아 (선택됨) ✅
   - 푸켓, 태국
   - 세부, 필리핀
   
   [탭 2: 항공권]
   - 대한항공, ICN → DPS
   - 2026-02-26 출발, 2026-03-03 귀국
   - 850,000원
   
   [탭 3: 일정]
   - Day 1 ~ Day 5 상세 일정
   
   [탭 4: 예산]
   - 항목별 예산 분배
   - 총 2,000,000원
   
   [사이드바: 실행 로그]
   - 각 에이전트의 실행 과정
   - RAG 검색 쿼리 및 결과
   - LLM 호출 및 응답 로깅
   ```

6. **사용자 만족도**
   ```
   ✅ 5분 만에 완성된 맞춤형 여행 계획
   ✅ 실시간 웹 정보 기반의 신뢰도 높은 추천
   ✅ 상세한 일정과 현실적인 예산 분배
   ✅ 투명한 프로세스 (로그 확인 가능)
   ```

#### **시나리오 2: 고층 빌딩 탐험 여행 계획**

**사용자**: 건축학도 박도시 (20대, 졸업 여행)

**상황**:
- 세계적인 고층 빌딩과 현대 건축물 탐방
- 예산 제한 없음 (부모님 지원)
- 사진 촬영이 주 목적

**서비스 이용 과정**:

1. **입력**: "높은 빌딩숲 탐험", 7일, 예산 미정
2. **Agent A**: 두바이, 뉴욕, 상하이 추천 → 두바이 선택
3. **Agent B**: 에미레이트 항공, 1,500,000원
4. **Agent C**: 
   - 부르즈 칼리파, 두바이 몰, 팜 주메이라 등 포함
   - 예산: 5,000,000원 (항공권 + 럭셔리 호텔 + 투어)

---

#### **시나리오 3: RAG 비활성화 테스트**

**사용자**: 개발자 이테스트 (서비스 테스트)

**입력**: RAG 체크박스 해제 ❌

**차이점**:
- **RAG ON**: 실시간 웹 검색 결과 기반 추천 (최신 정보, 구체적)
- **RAG OFF**: LLM 학습 데이터 기반 추천 (일반적, 빠름)

### **플로우 상세 설명**

#### **Phase 1: 사용자 입력 (User Input)**
```
시간: 0초
작업: 사용자가 Streamlit UI에서 폼 작성
데이터: 
  - travel_theme: "해변에서 휴양"
  - travel_days: 5
  - budget: 2000000
  - departure_city: "서울"
  - enable_rag: true
```

#### **Phase 2: Agent A 실행 (도시 추천)**
```
시간: 0-15초

[단계 1] 입력 로깅 (0.1초)
├─ 로거: CityRecommenderAgent
├─ 내용: 여행 주제, 일수, 예산 출력
└─ 위치: 콘솔 + Streamlit 로그

[단계 2] RAG 검색 (3-8초)
├─ 쿼리 생성: "해변에서 휴양 여행 추천 도시 해외"
├─ DuckDuckGo API 호출
│   ├─ 시도 1: 성공 또는 Rate Limit
│   ├─ 시도 2: 2초 대기 후 재시도
│   └─ 시도 3: 4초 대기 후 재시도
├─ 결과: 3개 웹페이지 (제목, 본문, URL)
└─ 컨텍스트 포맷팅: 약 1,500자

[단계 3] LLM 호출 (5-10초)
├─ 프롬프트 구성:
│   ├─ System: "당신은 전문 여행 컨설턴트입니다..."
│   └─ Human: "검색 결과: ... \n여행 주제: ..."
├─ OpenAI GPT-4 API 호출
├─ 응답: JSON 형식 (도시 2~3개)
└─ 토큰: 약 500 tokens

[단계 4] JSON 파싱 (0.1초)
├─ 코드 블록 제거 (```json```)
├─ JSON.loads()
├─ 결과 추출: recommended_cities
└─ 첫 번째 도시 자동 선택

[단계 5] 출력 로깅 (0.1초)
├─ 추출된 도시 목록 출력
├─ 선택된 도시 표시
└─ State 업데이트
```

#### **Phase 3: Agent B 실행 (항공권 검색)**
```
시간: 15-30초

[단계 1] 입력 로깅 + 날짜 계산 (0.1초)
├─ 선택된 도시 확인
├─ 출발일 = today + 30일
└─ 귀국일 = 출발일 + travel_days

[단계 2] RAG 검색 (3-8초)
├─ 쿼리: "서울 발리 항공권 평균 가격 항공사"
├─ DuckDuckGo 검색
└─ 항공권 가격 정보 수집

[단계 3] LLM 호출 (5-10초)
├─ 검색 결과 + 날짜 정보
├─ GPT-4가 항공권 정보 생성
└─ JSON: flight 객체

[단계 4] 파싱 + 로깅 (0.1초)
└─ State 업데이트
```

#### **Phase 4: Agent C 실행 (일정 계획)**
```
시간: 30-50초

[단계 1] 입력 로깅 (0.1초)
├─ 도시 + 항공권 정보 확인
└─ 여행 일수 확인

[단계 2] RAG 검색 (3-8초)
├─ 쿼리: "발리 해변휴양 여행 일정 추천 명소 맛집"
├─ 검색 결과 4개 (더 많은 정보 필요)
└─ 명소, 맛집, 액티비티 정보

[단계 3] LLM 호출 (10-15초)
├─ 가장 긴 프롬프트 (검색 결과 + 도시 + 항공권)
├─ GPT-4가 일정 + 예산 생성
└─ JSON: itinerary + budget_breakdown

[단계 4] 파싱 + 로깅 (0.1초)
├─ 일정 배열 추출
├─ 예산 객체 추출
└─ State.completed = true
```

#### **Phase 5: UI 렌더링**
```
시간: 50-52초

[Streamlit 렌더링]
├─ st.success("여행 계획이 완성되었습니다!")
├─ st.rerun() 호출
├─ 탭 UI 생성
│   ├─ 탭1: 추천 도시 (Expander)
│   ├─ 탭2: 항공권 (Metrics)
│   ├─ 탭3: 일정 (Expander)
│   └─ 탭4: 예산 (Metrics)
└─ 사이드바: 로그 (st.text)

[총 소요 시간]
약 50초 (RAG ON)
약 20초 (RAG OFF)
```

### **에러 처리 플로우**

```
┌─────────────────────────────────────────────┐
│  에러 발생 지점                              │
└──────────────┬──────────────────────────────┘
               │
               ├─ DuckDuckGo Rate Limit
               │  └─> 재시도 로직 (2회, 지수 백오프)
               │      ├─ 성공 → 계속 진행
               │      └─ 실패 → 빈 결과 반환 (RAG OFF 모드)

               ├─ JSON 파싱 오류
               │  └─> logger.error() + 빈 배열 반환
               │      └─> UI에 "정보 없음" 표시
               │
               ├─ LLM API 오류
               │  └─> 예외 처리 + 사용자에게 에러 메시지
               │
               └─ 네트워크 오류
                  └─> 타임아웃 + 재시도 + 에러 메시지
```

### **확장 가능성**

```
현재 구조 (MVP)
    │
    ├─ [Phase 2] 병렬 처리
    │   └─ Agent A → (Agent B || Agent C) 동시 실행
    │
    ├─ [Phase 3] 사용자 선택
    │   └─ 도시 선택 UI → Agent B, C 재실행
    │
    ├─ [Phase 4] 추가 에이전트
    │   ├─ Agent D: 숙소 추천 (Booking.com API)
    │   ├─ Agent E: 맛집 추천 (Google Places API)
    │   └─ Agent F: 날씨 정보 (OpenWeather API)
    │
    ├─ [Phase 5] 데이터베이스
    │   └─ 여행 계획 저장/불러오기 (SQLite → PostgreSQL)
    │
    └─ [Phase 6] 고급 RAG
        ├─ Vector DB (FAISS, Pinecone)
        ├─ 문서 임베딩 및 유사도 검색
        └─ 멀티소스 검색 (DuckDuckGo + Bing + Google)
```
